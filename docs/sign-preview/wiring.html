WLED setup (verified against WLED 16.x)

  1. Use 16.0.1 or newer — 16.0.1 fixed a ledmap parser bounds bug (reading past the end of the map array).
  2. Two controllers (or two outputs): board GPIO → 137 px, word GPIO → 459 px.
  3. Upload each map to the controller's filesystem at /edit, renamed to ledmap.json: wled/board-controller/ledmap.json — 41×55 grid, wled/word-controller/ledmap.json — 160×30 grid; 10 mm cells, −1 = empty cell. On reboot the map's "width"/"height" switch the controller into 2D matrix mode by itself (16.x deserializeMap sets the dims at boot) — 2D effects (plasma, Matrix, DNA…) then render in true sign space. Check Info shows the grid size. Never pretty-print these files: 16.x streams the map with a byte-exact "map":[ search — "map": [ (space) silently loads zero entries and the sign renders as if the chain were crammed into the top rows.
  4. No Config → 2D Configuration needed, and no 2d-gaps.json — gap files only patch panel-scan matrices and WLED ignores them whenever a custom ledmap exists. If the sign ever boots as a single line of pixels, open 2D Configuration and hit Save once (0.15-era quirk, issue #5082) — the ledmap re-wins on next boot.
  5. Segments become RECTANGLES once the map loads: start/stop = column range, startY/stopY = row range (stops exclusive), origin top-left. Chain-index segments (e.g. 87–107) would be read as columns — off-grid starts collapse to overlapping full-width strips, which looks like a few random cells lit. The shipped presets are rect stacks: bolt = full-grid yellow base + 5 red rectangles painted over it (v16 composites segments in id order, blend "top"); word = one column band per letter, cut at columns 25/54/77/106/132. 16 seam cells sit in the neighbor's band — invisible while all letters run one color. Upload each as presets.json at /edit (replaces all presets), or merge via Presets backup/restore. Trailing {"stop":0} entries clear any leftover segments from earlier experiments.
  6. Colors are software — the bolt's red zone is chain 87–107 no matter what's playing.
