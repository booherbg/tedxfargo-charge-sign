What is and isn't simulated

Bit-exact: the effect mode function, the geometry tables (charge_geometry.h), color math (color_fade ported line-for-line from WLED 16.0.1), 8-bit arithmetic, segment state (step/call/aux), the 42 fps service cadence, and the grid→LED routing (this page applies the committed ledmap.json exactly like WLED's customMappingTable).
Approximated: hw_random8() is a seeded PRNG (uniform like the ESP32's RNG, but a different — reproducible — sequence), LED glow/optics, and gamma/color response of the physical WS281x pixels.
Not simulated: stock WLED effects, palettes/transitions (v1 effects don't use them), brightness limiter, and the WLED UI layer.
