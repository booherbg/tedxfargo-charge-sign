What is and isn't simulated

Bit-exact: the effect mode functions, the geometry tables (charge_geometry.h), color math (color_fade/color_blend/ ColorFromPalette/color_from_palette ported line-for-line from WLED 16.0.1), the full palette system — WLED's own fastled_slim palette classes compiled verbatim, palette data extracted from the firmware source (never transcribed), your device's custom palettes, the 8 CHARGE usermod palettes at the same IDs the firmware registers, and color-derived palettes from the segment colors — 8-bit arithmetic, segment state and all params (custom3 is 5-bit like WLED), the 42 fps service cadence, and the grid→LED routing (the committed ledmap.json applied exactly like WLED's customMappingTable).
Approximated: hw_random8() is a seeded PRNG (uniform like the ESP32's RNG, but a different — reproducible — sequence); palette "* Random Cycle" regenerates from a hash instead of WLED's harmonic random walk; audio here is a synthetic beat or your browser microphone, while the sign uses its onboard mic through the audioreactive usermod (same 0..255 level + peak interface on both sides); LED glow/optics.
Not simulated: stock WLED effects, palette/effect transitions, brightness limiter, WLED UI.
